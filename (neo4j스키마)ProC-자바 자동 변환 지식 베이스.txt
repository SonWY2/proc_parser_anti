📊 개요
이 그래프는 Pro*C(C 언어에 내장된 SQL) 프로그램을 Java로 자동 변환하기 위한 지식 베이스입니다.
현재 통계:
노드 타입: 13개
관계 타입: 18개
프로그램: 약 40-50개
전체 노드: 약 10,000-50,000개 (추정)
🏷️ 주요 노드 타입 (13개)
Program: 프로그램 루트 노드
Function: C 함수
ProCVariable: Pro*C 변수
JavaVariable: Java 변수 (camelCase 변환)
StructDefinition: C 구조체
FieldDefinition: 구조체 필드
OMMFile: DTO 클래스
OMMField: DTO 필드
DBIOFile: MyBatis Mapper
DBIOQuery: SQL 쿼리
DaoFile: DAO 인터페이스
DaoMethod: DAO 메서드
ConversionRule: 변환 규칙
상세 속성 및 관계는 영문 문서 참조
🔗 주요 관계 타입 (18개)
프로그램 소유권
CONTAINS: Program → Function
DECLARES: Program → ProCVariable
DEFINES: Program → StructDefinition
HAS_OMM: Program → OMMFile
HAS_DBIO: Program → DBIOFile
HAS_DAO: Program → DaoFile
함수 관련
CALLS: Function → Function (호출)
USES_VARIABLE: Function → ProCVariable
USES_STRUCT_TYPE: Function → StructDefinition
USES_CONVERSION: Function → ConversionRule
변수/타입 매핑
MAPS_TO: ProCVariable → JavaVariable
HAS_TYPE: ProCVariable → StructDefinition
HAS_FIELD: StructDefinition → FieldDefinition
CORRESPONDS_TO: JavaVariable → FieldDefinition (⭐ 2025-01-17 수정)
CORRESPONDS_TO: JavaVariable → OMMField (⭐ 2025-01-17 수정)
DAO/DBIO
CONTAINS_QUERY: DBIOFile → DBIOQuery
IMPLEMENTS: DBIOFile → DaoFile
IMPLEMENTS: DBIOQuery → DaoMethod
USES_INPUT_DTO: DBIOQuery/DaoMethod → OMMFile
USES_OUTPUT_DTO: DBIOQuery/DaoMethod → OMMFile
